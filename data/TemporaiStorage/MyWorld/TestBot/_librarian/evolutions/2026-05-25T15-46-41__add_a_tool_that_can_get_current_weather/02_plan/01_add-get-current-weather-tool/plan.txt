GOAL
Add a `get_current_weather(location)` tool to TestBot that fetches live weather data
from the wttr.in public JSON API (no API key, stdlib-only HTTP).

DEPENDS ON
- src/agents/TestBot/main.py (existing tool file)
- No new dependencies — uses urllib.request (stdlib)

CHANGES
File: src/agents/TestBot/main.py

1. Add import for `urllib.request`, `urllib.parse`, `urllib.error` at top
   (all stdlib, already available).

2. Add the tool function after `get_current_datetime` and before `evolve_capability`:

    @tool([AGENT_NAME])
    def get_current_weather(self: Mind, location: str = "") -> str:
        """Return current weather conditions for a given location.

        Use when: the user asks about the weather, temperature, conditions,
        humidity, or wind for a specific city, region, or place.
        Do NOT use for historical weather or multi-day forecasts.

        location: City or place name (e.g. "London", "New York", "Tokyo").
        """
        if not location:
            return "Error: 'location' parameter is required. Provide a city or place name."

        encoded = urllib.parse.quote(location)
        url = f"https://wttr.in/{encoded}?format=j1"

        try:
            req = urllib.request.Request(url, headers={"User-Agent": "TestBot/1.0"})
            with urllib.request.urlopen(req, timeout=10) as resp:
                data = json.loads(resp.read().decode())
        except urllib.error.URLError as exc:
            return json.dumps({"error": f"Could not reach weather service: {exc.reason}"})
        except Exception as exc:
            return json.dumps({"error": str(exc)})

        current = data.get("current_condition", [{}])[0]
        desc = current.get("weatherDesc", [{}])[0].get("value", "Unknown")
        nearest = data.get("nearest_area", [{}])[0]
        area = nearest.get("areaName", [{}])[0].get("value", location)
        country = nearest.get("country", [{}])[0].get("value", "")

        return json.dumps({
            "location": f"{area}, {country}" if country else area,
            "condition": desc,
            "temperature_c": current.get("temp_C", "?"),
            "temperature_f": current.get("temp_F", "?"),
            "feels_like_c": current.get("FeelsLikeC", "?"),
            "feels_like_f": current.get("FeelsLikeF", "?"),
            "humidity_pct": current.get("humidity", "?"),
            "wind_kmph": current.get("windspeedKmph", "?"),
            "wind_direction": current.get("winddir16Point", "?"),
            "visibility_km": current.get("visibility", "?"),
            "uv_index": current.get("uvIndex", "?"),
        })

3. Expose on Mind and add to TOOLS registry:
    Mind.get_current_weather = get_current_weather   # after existing Mind.get_current_datetime line
    TOOLS = [greet, get_today_date, get_current_datetime, get_current_weather, evolve_capability]

CONSTRAINTS
- stdlib HTTP only (urllib) — no requests, httpx, or new packages
- The tool must handle network errors gracefully (return JSON error, not raise)
- Empty location returns a plain error string (consistent with evolve_capability pattern)
- timeout=10 seconds to avoid hanging the agent

RISKS
- wttr.in is a public free API with no uptime guarantees; if it's down, tool returns a JSON error dict
- Rate limits possible under heavy load; not a concern for test usage
- The API may resolve ambiguous city names differently than expected

ROLLBACK
Remove the `get_current_weather` function, its Mind exposure line, and revert
the TOOLS list to its previous state. No schema migrations or external state changes.
